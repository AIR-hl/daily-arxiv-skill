# 写作风格

## 核心要求

- 最终结果必须是一篇连续的中文精读笔记，不是若干小节各自交差。
- 文章要回答四个问题：它想补什么空白、它到底怎么工作、证据够不够、有什么启发洞见。
- 除非用户明确要求其他语言，默认使用简体中文。

## 机制图与公式

- 方法机制优先使用"机制图 + 解释 + 承重类比"的连贯方式，让读者能一路跟下来。
- 图示可以使用 ASCII，必要时用代码块排版；如果原文图示表达清晰对理解有帮助，请插入论文原图。
- 请保留核心公式，有时候一式胜千言；公式请放到公式块中，不要用反引号伪装。

## 审稿口径

在"博导审稿"节中，除模板占位符已要求的判断外，还需额外覆盖：

- 选题是真空白，还是人为制造出的空白
- 写作是否把真正的解释负担藏起来了

## 成稿规范

草稿完成后做一轮收尾检查：

- 如果最终保存目标是飞书文档，不要在本文件里临时决定组件策略；保存前读取 `references/lark-rendering.md`，由 Render 阶段集中处理飞书组件化。
- `不是...而是...`、`不是...是...`、`不再是...而是...` 这种句式全文最多出现两次。
- 明显机械的三项并列，优先改成两项或四项。
- 同一段里破折号超过两次时，请进行句式调整，多余部分改成逗号或句号。
- 如果增量很薄，就直说薄，实事求是不要硬吹。
