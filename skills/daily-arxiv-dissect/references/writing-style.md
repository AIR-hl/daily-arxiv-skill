# 写作风格

这个文件只在单篇精读实际写作阶段读取，用来约束文章节奏和细节表达。

## 核心要求

- 最终结果必须是一篇连续的中文精读文章，不是若干小节各自交差。
- 文章要回答四个问题：它想补什么空白、它到底怎么工作、证据够不够、我最后怎么判断。
- 除非用户明确要求其他语言，默认使用简体中文。

## 推荐写作节奏

可以围绕下面的运动方向组织文章，但不要写成僵硬的 checklist：

1. 研究空白与问题来龙去脉
2. 真实增量是什么
3. 方法机制图与承重类比
4. 读懂全文必须抓住的关键概念
5. 关键技术细节与证据强度
6. 导师式审稿判断与 takeaway

## 机制图与公式

- 图示默认使用 ASCII，必要时用代码块排版。
- 只允许使用纯 ASCII 字符：`+ - | / \ > < v ^ * = ~ . : # [ ] ( ) _ , ; ! ' "`
- 不要使用 Unicode 线框字符或箭头字符。
- 如果核心公式对理解机制必不可少，就保留；公式使用 `$$...$$`，不要用反引号伪装。

## 审稿口径

至少覆盖下面这些问题：

- 选题是真空白，还是人为制造出的空白
- 方法是优雅推进，还是蛮力堆料
- 实验是否公平，消融是否诚实，结论是否经得起细看
- 写作是否把真正的解释负担藏起来了
- 最后给出一句明确判决：`strong accept`、`weak accept`、`borderline`、`weak reject`、`strong reject`

## 文风清洁

草稿完成后，做一轮轻量清洁：

- `不是...而是...`、`不是...是...`、`不再是...而是...` 全文最多出现两次
- 明显机械的三项并列，优先改成两项或四项
- 同一段里破折号超过两次时，多余部分改成逗号或句号
- 如果增量很薄，就直说薄，不要硬吹
