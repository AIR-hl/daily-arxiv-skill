# 飞书渲染

这个文件只在论文精读报告需要保存到飞书文档时读取。

## 三阶段流程

### Compose

- 先根据 `assets/templates/paper_template.md` 组织论文精读内容，保持模板定义的章节顺序和写作责任。
- Compose 阶段的目标是内容正确、论证完整、结构清晰；它不是最终飞书阅读形态。
- 不要因为后续要做飞书渲染而改变模板的核心章节结构，也不要把报告写成 checklist。

### Render

- 在执行飞书文档创建或更新前，把 Compose 后的内容转换为适合飞书阅读的发布稿。
- Render 阶段采用自适应策略：根据论文内容选择能提高阅读效率的飞书呈现方式，不固定每个章节必须使用的组件，也不要为了使用组件而破坏文章节奏。
- 如果使用飞书扩展能力，请参考相关 SKILL 说明，不要在凭记忆硬写具体语法。

### Publish

- 发布到飞书时，传给飞书文档创建或更新操作的应是面向飞书阅读体验处理过的内容，不要把未经任何阅读体验审视的模板化正文直接发布。

## 渲染目标

排版渲染的终极目标是**降低认知负担，提升扫读效率**。在将正文转换为飞书发布稿时，请遵循以下原则进行渲染优化：

### 1. 建立视觉焦点（引导扫读路径）

- **高亮关键论断**：让动机、论断、局限、证据强弱级别的犀利判断在视觉上跳脱出来。对“实验证据不足”“窄场景成立”“baseline 不公平”等结论，应果断使用**加粗**、颜色文字或高亮块处理。
- **及时分点分段**：避免长篇大论的文字墙。章节内的分点论述应拆分为带序号的段落，并对每个小点的**核心句或关键词进行加粗**，确保读者仅看加粗部分即可掌握段落骨架。

### 2. 信息结构降维（辅助快速定位）

- **概念卡片化**：让关键定义、作者原始立场、重要假设更容易被检索。应将这些核心概念提取为“引用样式（Blockquote）”或专门的高亮说明块。
- **对比表格化**：切忌用大段文字堆砌对比关系。当遇到多个方法比较、实验设置、消融结果或并列的风险点时，**优先转化为表格**，利用矩阵视图直观展现差异。

### 3. 灵活应用图表组件（表达复杂逻辑）

- **组件化流程**：当需要描述机制流程、模块关系或因果链条时，鼓励使用飞书丰富的图表组件。**可以灵活运用画板、流程图或 ASCII 图**来重新梳理逻辑，替代冗长的纯文本描述。

### 4. 文本元素规范化（注重专业细节）

- **专有名词隔离**：段落中出现的数据集（如 `SWE-Bench`）、模型名称（如 `Qwen3`）、关键数据，统一使用**行内代码**样式，使其与普通中文字符产生良好的边界感与阅读节奏。
- **公式严谨排版**：所有公式、变量的latex语法必须正确，并且包裹在独立的**公式块**或**行内公式**中，不要使用代码样式并杜绝源码裸露。

### 5. 核心心法：保持克制

- **避免过度包装**：组件的使用是为了提升阅读效率，而不是为了炫技。在利用排版打破大段文字沉闷感的同时，**切忌为了“看起来丰富”而滥用复杂组件**。保持页面整洁，让排版自然地服务于内容理解即可，无需过度设计。

## 发布前检查

- 检查最终发布稿是否完整满足 `paper-template` 定义的章节与写作责任，比如：论文元信息、动机、前置认知、方法剖析、关键细节、博导审稿、总结与洞见。
- 检查最终发布稿是否经过飞书阅读场景下的可读性审视，包括层次、重点、图示、对比和扫读路径。
- 如果最终发布稿接近纯文本也不是不行，但要先确认这是出于内容需要，而不是遗漏了 Render 阶段。
