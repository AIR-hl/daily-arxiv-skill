# 筛选策略

这个文件存放“每日 arXiv”任务的默认筛选偏好。只有在父 skill 需要排序、取舍或解释为什么选它不选它时才读取。

## 边界

父 skill 在筛选阶段通常只能看到标题、摘要、comment、分类、关键词命中和来源链接，不需要阅读完整论文正文。

因此这里的规则只用于**摘要级初筛**，不是对论文真实增量或实验可信度的最终判决。真正的“增量质量”判断必须由完整阅读论文的 `daily-arxiv-dissect` 子 skill 完成。

## 排序原则

首先查看同目录下的 `description.md`（即 `references/description.md`），这是用户对自己工作内容的自述，然后通过摘要内容判断的论文价值，最后看组织来源。

### 1. 文章推测质量

这里不是判断"真实增量是否成立"，而是根据摘要和元数据推测文章是否值得送去深读。

**首要权重：与用户的相关性。** 优先选择对用户真实任务与工作实践有潜在可迁移性或直接启发的论文；

**正向信号**
- 摘要明确说出了"以前做不到什么，现在多做成了什么"。
- 摘要给出具体方法线索，而不是只有空泛口号。
- comment、标题或摘要里出现强可信信号，例如SOTA、会议投稿信息。

**负向信号（可直接排除）：**
- 明显像"只换包装的热点复述"：新造概念讲故事，重复之前工作相同或相似的流程，没有新机制。
- 纯理论研究，没有落地实践的可能

### 2. 组织来源

文章的来源也是一项重要的参考信息，通常知名高校以及公司的论文可能具有更高的潜在价值：

- 大厂研究团队，例如 ByteDance Seed、Alibaba Tongyi、Ant、Kuaishou、Google、GLM
- 强研究高校或研究组，例如 Princeton、Stanford、Tsinghua、Fudan、Beihang

只有在信息可核对时才使用组织来源，不要根据作者名字臆测 affiliation。

## 例外情况

- 如果论文不完全属于主线，但实践价值明显更高，可以入选；父日报里要解释“为什么破例”。
- 如果候选池整体质量一般，也不要为了凑数硬选。宁可少选几篇，也不要把噪声送去深读。
