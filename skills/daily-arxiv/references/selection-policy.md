# 筛选策略

这个文件存放“每日 arXiv”任务的默认筛选偏好。只有在父 skill 需要排序、取舍或解释为什么选它不选它时才读取。

## 边界

父 skill 在筛选阶段通常只能看到标题、摘要、comment、分类、关键词命中和来源链接，拿不到完整论文正文。

因此这里的规则只用于**摘要级初筛**，不是对论文真实增量或实验可信度的最终判决。真正的“增量质量”判断必须由完整阅读论文的 `daily-arxiv-dissect` 子 skill 完成。

在执行这些规则时：

- 不要下载 PDF
- 不要打开正文
- 不要读取 `pdf_url` 或 `source_url`
- 不要做“轻量精读核对”

## 默认关注方向

优先关注能直接服务下面这些工作主题的论文：

- Agent
- RL
- Reward Modeling
- Data Synthesis
- DPO 及其相关问题
- 智能客服工作实践有关的主题

## 排序原则

默认先看相关性，再看摘要里暴露出的信号质量，最后看组织来源。

### 1. 相关性

- 摘要是否明显命中当前关注方向
- 是否对真实工程问题有可迁移的启发
- 是否只是“蹭热点术语”，但内容离主线很远

### 2. 摘要信号质量

这里不是判断“真实增量是否成立”，而是看摘要和元数据里有没有值得送去深读的信号：

- 摘要是否清楚说出“以前做不到什么，现在多做成了什么”
- 摘要是否给出具体的方法线索，而不是只有空泛口号
- comment、标题或摘要里是否出现值得追的强信号，例如新 benchmark、明确方法名、清晰任务设定、开源承诺、会议投稿信息等
- 是否明显像“只换包装的热点复述”或“几乎全靠堆模型/堆数据”的工作；如果是，优先级下降

父 skill 可以据此做送审优先级排序，但不要把这一步写成对论文贡献已经被验证的结论，也不要为了“更稳”而越权去读全文。

### 3. 组织来源

如果前两项相近，再用组织来源做 tie-break。默认偏好：

- 大厂研究团队，例如 ByteDance Seed、Alibaba Tongyi、Ant、Kuaishou、Google、GLM
- 强研究高校或研究组，例如 Princeton、Stanford、THU、ZJU、FDU

只有在信息可核对时才使用组织来源，不要根据作者名字臆测 affiliation。

## 例外情况

- 如果论文不完全属于主线，但实践价值明显更高，可以入选；父日报里要解释“为什么破例”。
- 如果候选池整体质量一般，也不要为了凑数硬选 4 篇。宁可选 1-2 篇，也不要把噪声送去深读。
