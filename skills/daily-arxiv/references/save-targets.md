# 保存目标

这个文件只在需要把日报或单篇精读落到飞书知识库时读取。

## 目标解析规则

- 只使用知识库模式；如果目标不是知识库节点、知识库 URL 或 `wiki_space`，就停止并确认，不要自行切换到其他存储方式。
- 如果用户要求保存但没有单独指定目标，默认按名为 `日常学习` 的知识库、节点标题为 `每日Arxiv` 解析落点；解析失败时向用户确认，不要自行降级。
- 如果用户给出了知识库名称，按该知识库的根节点作为落点。
- 如果用户给出了知识库名称 + 节点标题（或路径），按该节点作为落点。
- 如果用户给出了 `wiki_node`、知识库 URL 或 `wiki_space`，直接按对应节点解析。
- 落点确定后，父日报创建为该节点的直接子文档；子精读再创建为父日报的子文档，不受落点层级影响。
- 如果用户只口头描述了目标位置，优先围绕知识库名称、节点标题做一次轻量定位；仍无法确定时，向用户索要更明确的目标。
- 不要把普通 `/docx/...` 文档误当作可继续挂子页面的知识库节点。

## 父日报规则

- **父日报标题必须严格基于任务执行当天日期**，格式为 `YYYY-MM-DD日报`，例如 `2026-04-11日报`。禁止添加其他任何前缀、后缀或说明文字。
- 标题放在 `--title`，正文不要再重复同名一级标题。
- 父日报正文必须基于 `assets/templates/summary_template.md` 组织，而不是随机组织文章结构。

## 父 -> 子落点传递

父 skill 在创建完父日报后，应把解析好的落点和父日报引用一起传给子 skill。

- 知识库模式：传 `save_mode=wiki`。
- 无保存模式：传 `save_mode=none`，且不创建飞书文档。

同时把 `parent_doc` 传给子 skill，作为父日报引用；子 skill 据此确定子文档的保存位置，至少包含：
- `doc_url`
- `doc_id`
- `title`

## 成功回链

子 skill 保存成功后，父 skill 应把子文档链接回填到父日报对应论文条目里，并在最终回复中说明：

- 父日报链接
- 每篇子精读链接
- 如果发生权限补齐，说明最终状态

回填父日报时，链接文本优先使用子 skill 返回的 `doc_title`，避免只留下裸 URL。
